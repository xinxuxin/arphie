# AI Tool Notes

AI coding assistants helped draft the project scaffold, create small implementation steps, suggest tests, and keep the CLI and web app connected to the same shared engine.

They are useful for repetitive wiring, documentation drafts, test case ideas, and spotting edge cases such as empty folders, unsupported files, and missing indexes.

They can be wrong when they assume APIs behave a certain way, miss dependency details, or generate code that looks plausible but is not actually exercised.

Verification is still required. The project should be checked with automated tests, manual CLI runs, manual web API smoke tests, and review of source citations to confirm answers come from retrieved chunks.
